## 2024-05-18 - [Fix Hardcoded Legacy Password Fallback]
**Vulnerability:** In `server.js`, creating user profiles via the admin panel defaulted to a predictable and easily guessable plaintext password `'123456'` if no password was provided. This fallback occurred implicitly because a `passwordHash` was not saved, triggering fallback logic upon subsequent login attempts.
**Learning:** Hardcoded fallback credentials bypass proper security provisioning and lead to easily exploitable accounts. Furthermore, introducing a hardcoded string as a fallback argument to `bcrypt.hash()` during account creation is equally problematic.
**Prevention:** Avoid relying on hardcoded defaults. Always strictly generate secure fallback passwords using `crypto.randomBytes(16).toString('hex')` and properly hash them via `bcrypt` to ensure the stored credentials in the database cannot be easily guessed. Always verify all properties are supported by the `schema.prisma`.
